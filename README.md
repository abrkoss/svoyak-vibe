# Своя игра (локальная WiFi)

Веб-приложение для проведения игры типа «Своя игра» в локальной WiFi-сети.
Бэкенд — Node.js (Express + Socket.IO), фронтенд — Vue 3 (Vite). Один сервер отдаёт
собранный фронт и обслуживает realtime на одном порту.

## Три экрана

- `/display` — экран-показ (проектор): табло и текущий вопрос **без ответа**.
- `/host` — ведущий: вопрос **с ответом**, выбор вопроса, приём кнопок, подтверждение
  верно/неверно, управление раундами и финалом.
- `/player` — игрок (телефон): кнопка «Ответить» (гонка), в финале — ставка и текстовый
  ответ.

## Установка

```bash
npm install
```

## Контент игры

Вопросы берутся из `data/questions.json`. Отредактируйте его перед игрой
(за образец — `data/questions.example.json`). Структура:

```json
{
  "title": "Название игры",
  "rounds": [
    {
      "name": "Раунд 1",
      "type": "normal",
      "themes": [
        { "name": "Тема", "questions": [
          { "value": 100, "text": "Вопрос", "answer": "Ответ" }
        ] }
      ]
    },
    {
      "name": "Финал",
      "type": "final",
      "themes": [ { "name": "Тема", "text": "Вопрос", "answer": "Ответ" } ]
    }
  ]
}
```

- Обычные раунды: `type: "normal"`, у каждой темы список `questions` с ценой `value`.
- Финальный раунд: `type: "final"`, у каждой темы один вопрос (без `value`).

## Запуск

### Боевой режим (для игры)

```bash
npm run build   # собрать фронтенд в dist/
npm start       # запустить сервер
```

При старте сервер печатает адреса для подключения, например:

```
Играем! Откройте на устройствах:
  Экран-показ:  http://192.168.0.10:3000/display
  Ведущий:      http://192.168.0.10:3000/host
  Игроки:       http://192.168.0.10:3000/player
```

Все устройства должны быть в одной WiFi-сети. Игроки открывают адрес `/player`,
вводят имя и попадают в лобби.

### Режим разработки

```bash
npm run dev
```

Поднимает Node-сервер (порт 3000) и Vite dev-сервер (порт 5173, проксирует socket.io).

## Ход игры

1. Игроки заходят на `/player`, вводят имена — ведущий видит их в лобби.
2. Ведущий кликает клетку табло → вопрос появляется на показе.
3. Ведущий жмёт «Открыть кнопки» → игроки могут нажать «Ответить».
4. Кто первый нажал — отвечает устно; ведущий жмёт «Верно» (+цена) или «Неверно»
   (−цена, кнопки открываются для остальных).
5. Когда раунд сыгран — «Следующий раунд».
6. В финале ведущий удаляет темы до одной, игроки делают скрытые ставки, отвечают
   текстом, ведущий вскрывает и подтверждает ответы.

Счёт сохраняется в `data/game-state.json` — перезапуск сервера не теряет прогресс.
Чтобы начать новую игру с нуля, удалите этот файл.
