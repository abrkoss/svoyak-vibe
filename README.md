# Своя игра (локальная WiFi)

Веб-приложение для проведения игры типа «Своя игра» в локальной WiFi-сети.
Бэкенд — Node.js (Express + Socket.IO), фронтенд — Vue 3 (Vite). Один сервер отдаёт
собранный фронт и обслуживает realtime на одном порту.

## Три экрана

- `/display` — экран-показ (проектор): табло и текущий вопрос **без ответа**.
- `/host` — ведущий: вопрос **с ответом**, выбор вопроса, приём кнопок, подтверждение
  верно/неверно, управление раундами и финалом.
- `/player` — игрок (телефон): кнопка «Ответить» (гонка), в финале — ставка и текстовый
  ответ.

## Установка

```bash
npm install
```

## Контент игры

Игры лежат в `data/games/<id>/game.json`. Каждая папка — отдельная игра. Ведущий
выбирает игру из списка при старте. Образец структуры — `data/questions.example.json`
и демо-игра `data/games/demo/game.json`.

```json
{
  "title": "Название игры",
  "settings": {
    "buzzerDelaySec": 1,
    "buzzerTimeSec": 30,
    "answerTimeSec": 30
  },
  "rounds": [
    {
      "name": "Раунд 1",
      "type": "normal",
      "themes": [
        { "name": "Тема", "questions": [
          {
            "value": 100,
            "text": "Вопрос",
            "answer": "Ответ",
            "buzzerTimeSec": 45,
            "media": {
              "image": "media/photo.jpg",
              "audio": "media/clip.mp3",
              "video": "media/clip.mp4"
            },
            "answerMedia": {
              "image": "media/answer-photo.jpg",
              "video": "media/answer-clip.mp4"
            }
          }
        ] }
      ]
    },
    {
      "name": "Финал",
      "type": "final",
      "themes": [ { "name": "Тема", "text": "Вопрос", "answer": "Ответ" } ]
    }
  ]
}
```

- Обычные раунды: `type: "normal"`, у каждой темы список `questions` с ценой `value`.
- Финальный раунд: `type: "final"`, у каждой темы один вопрос (без `value`).
- Медиафайлы кладите в `data/games/<id>/media/`; пути в JSON — относительно папки игры.
  Поля `media.image`, `media.audio`, `media.video` опциональны (медиа вопроса).
  Поле `answerMedia` — отдельные медиа для показа правильного ответа на экране.
- Таймеры настраиваются в `settings`: `buzzerDelaySec` (задержка до кнопок, по умолчанию 1),
  `buzzerTimeSec` (время на нажатие кнопки, 30), `answerTimeSec` (время на ответ, 30).
  У отдельного вопроса можно задать свой `buzzerTimeSec` — он переопределит значение из `settings`.
- Игроки могут нажать **Пас** пока открыты кнопки; если все нажали пас — вопрос закрывается.

Чтобы добавить новую игру, создайте папку `data/games/my-game/` с файлом `game.json`.

## Запуск

### Боевой режим (для игры)

```bash
npm run build   # собрать фронтенд в dist/
npm start       # запустить сервер
```

При старте сервер печатает адреса для подключения, например:

```
Своя игра запущена!
  Экран-показ:  http://192.168.0.10:3000/display
  Ведущий:      http://192.168.0.10:3000/host
  Игроки:       http://192.168.0.10:3000/player
```

Все устройства должны быть в одной WiFi-сети. Игроки открывают адрес `/player`,
вводят имя и ждут, пока ведущий выберет игру.

### Режим разработки

```bash
npm run dev
```

Поднимает Node-сервер (порт 3000) и Vite dev-сервер (порт 5173, проксирует socket.io и `/games`).

## Ход игры

1. Ведущий открывает `/host` и выбирает игру из каталога.
2. Игроки заходят на `/player`, вводят имена — ведущий видит их в лобби.
3. В лобби ведущий может удалить игрока (кнопка ✕).
4. Ведущий кликает клетку табло → вопрос появляется на показе; через 1 с кнопки открываются автоматически.
5. У игроков есть таймер на нажатие кнопки; если никто не нажал — вопрос закрывается.
6. Отвечающему игроку отводится таймер; по истечении ведущий видит предупреждение и сам решает, верно или нет.
7. Кто первый нажал — отвечает устно; ведущий жмёт «Верно» (+цена) или «Неверно»
   (−цена, кнопки снова открываются для остальных). После верного ответа, если все пасуют
   или истекает таймер кнопки — на экране показывается правильный ответ, затем ведущий
   возвращается к табло.
8. Когда раунд сыгран — «Следующий раунд».
9. В финале ведущий удаляет темы до одной, игроки делают скрытые ставки, отвечают
   текстом, ведущий вскрывает и подтверждает ответы.

**Управление игрой (ведущий):**
- **Перезапустить** — та же игра с нуля (счёт и табло сбрасываются, игроки остаются).
- **Выйти** — вернуться к выбору игры.

Счёт сохраняется в `data/game-state.json` — перезапуск сервера не теряет прогресс.
Чтобы начать полностью с нуля, удалите этот файл.
